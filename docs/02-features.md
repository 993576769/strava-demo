# 功能模块

## 目标

拆解当前产品的核心功能，明确 MVP 范围与后续增强方向。

## 当前结论

- 当前 MVP 必做功能：
  - GitHub OAuth 登录与会话恢复
  - Strava 连接、断开、状态读取
  - 活动同步和活动列表展示
  - 活动详情与轨迹展示
  - 生成任务创建、底稿上传和结果展示
  - 生成历史记录和下载
  - Prompt 模板管理（管理员）
  - Sync event 调试面板
- 当前后端已具备的接口域：
  - `auth`
  - `integrations/strava`
  - `activities`
  - `art`
  - `sync-events`
- 当前后端已具备的基础能力：
  - JWT access token + refresh token
  - PostgreSQL 持久化
  - S3 文件上传
  - 单进程内任务轮询器

## 后续增强功能

- 邮箱密码注册、登录、找回密码
- 结果分享页和公开链接
- 结果再生成和更多模板
- 多平台活动来源
- 更丰富的任务调度和并发控制

## 模块边界建议

- 账号模块：产品身份、会话、权限
- Strava 模块：授权、活动同步、webhook
- 活动模块：列表、详情、轨迹读取
- 生成模块：任务、底稿、执行器、结果
- 作品模块：回看、下载、分享扩展

## 待细化

- 管理员能力是否继续放在同一个前端应用中。
- 生成模板是继续走数据库管理，还是引入版本化配置。
- 是否增加用户级生成额度和限流。
