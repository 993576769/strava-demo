# 功能模块

## 目标

拆解产品首批核心功能，明确 MVP 范围与后续增强方向，避免早期范围蔓延。

## 当前结论

- 账号体系采用分阶段建设：
  - 当前 MVP 只交付 GitHub 授权登录。
  - 邮箱/账号密码注册、账号密码登录、找回密码、重置密码放入后续增强阶段。
  - 接口和数据模型提前按多认证方式兼容设计，但首版页面和交互不一次性铺开。
- MVP 必做功能：
  - GitHub 授权登录与基础身份管理。
  - Strava OAuth 授权连接。
  - 活动同步与活动列表展示。
  - 轨迹活动详情查看。
  - 手绘轨迹图生成任务发起与结果查看。
  - 生成历史记录与基础下载能力。
- MVP 可选但优先级次高的功能：
  - 生成风格模板选择。
  - 再生成与参数微调。
  - 同步状态提示与失败重试入口。
- 后续增强功能：
  - 邮箱/账号密码注册与登录。
  - 找回密码、重置密码和账号恢复流程。
  - 分享页面或公开作品页。
  - 多种导出比例与海报模版。
  - AI 文案、标题、日期排版增强。
  - 收藏、标签、作品合集。
  - 支持更多运动平台而不只限于 Strava。
- 模块边界建议：
  - 账号模块负责当前 GitHub 登录态，并为后续账号密码体系预留统一会话结构。
  - 同步模块负责“拿到可靠活动数据”。
  - 生成模块负责“把活动数据转成图像任务和结果”。
  - 作品模块负责“管理已生成的成品资产”。

## 待细化

- MVP 是否必须包含分享能力。
- 账号密码体系是否优先支持“邮箱注册”，还是支持“用户名或邮箱登录”。
- 免费额度、生成次数限制、会员体系是否纳入首版。
- 风格模板数量与默认模板策略。
- 是否需要手动上传 GPX 作为非 Strava 入口。
