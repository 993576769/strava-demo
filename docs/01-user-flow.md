# 用户流程

## 目标

梳理用户从首次进入产品，到同步活动、生成轨迹图，再到查看结果的主路径。

## 当前结论

- 主路径分两段：
  - 先完成产品账号登录。
  - 再完成 Strava 连接、活动同步、图片生成和结果查看。
- 当前 MVP 主流程：
  1. 用户进入首页或登录页。
  2. 用户点击 GitHub 登录。
  3. 服务端完成 GitHub OAuth 回调，签发 access token 和 refresh token。
  4. 前端通过 `GET /api/auth/session` 恢复当前用户。
  5. 未连接 Strava 的用户进入连接引导。
  6. 用户连接 Strava，服务端保存连接并允许后续同步。
  7. 用户在活动列表查看本地同步结果。
  8. 用户进入活动详情页，查看轨迹和可生成状态。
  9. 用户选择模板并创建生成任务。
  10. 用户上传路线底稿，任务进入后台执行。
  11. 用户查看结果详情并下载图片。

## 当前 MVP 用户路径

### 路径 A：新用户首次进入

1. 使用 GitHub 登录。
2. 登录成功后进入产品壳层。
3. 如果未连接 Strava，则看到连接引导。
4. 连接后手动触发同步。
5. 同步完成后进入活动列表。

### 路径 B：已连接用户日常使用

1. 再次进入产品并恢复登录态。
2. 前端读取当前会话和 Strava 连接状态。
3. 用户查看活动列表和历史生成记录。
4. 用户选择活动并创建新任务。

### 路径 C：活动到成品

1. 用户打开活动详情。
2. 页面展示活动摘要、轨迹预览、模板选项和已有任务。
3. 用户提交生成任务。
4. 页面轮询任务状态。
5. 结果生成后跳转结果页或从列表打开。

## 异常与分支流程

- GitHub 登录失败：回到登录页，提示 OAuth 配置或回调失败。
- 未连接 Strava：优先展示连接引导，而不是空活动列表。
- Strava 连接失效：提示重新授权或断开连接。
- 同步失败：展示同步事件和最近错误信息。
- 活动不可生成：展示 `generatable_reason`。
- 生成失败：保留失败任务和错误信息，允许重试。
