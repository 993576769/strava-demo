# 风险与待决问题

## 目标

收敛当前项目推进中最值得提前关注的风险和未决策事项，避免在实现中期才暴露关键阻塞。

## 当前结论

- 技术风险：
  - Strava API 配额、授权审核和速率限制可能影响同步体验。
  - 长轨迹或大量活动数据可能带来存储和处理压力。
  - AI 图像生成成本、时延和稳定性存在不确定性。
  - 异步任务体系如果设计过轻，后续可能难以扩展。
  - 当前先上 GitHub 登录、后续再补账号密码体系，如果会话模型设计不统一，后续扩展成本会升高。
- 产品风险：
  - 用户是否真的愿意为“运动轨迹艺术化”能力持续使用或付费，仍需验证。
  - 风格化结果如果不稳定，容易影响第一印象。
  - 如果首版只能下载、不能方便分享，传播性可能不足。
- 合规与平台风险：
  - Strava 数据使用与展示需要遵守平台政策。
  - 用户地理轨迹涉及隐私，公开分享时可能需要脱敏能力。
  - 调用第三方 AI 服务时，需要考虑数据传输和服务条款。
- 当前开放问题：
  - 账号密码体系上线时，是否允许同一用户同时绑定 GitHub 和邮箱密码两种登录方式。
  - 首版是否只面向 Strava，还是预留多平台接入接口。
  - 手绘风格是否提供多个模板，还是先只做一种高质量默认风格。
  - 生成结果是偏艺术海报，还是偏真实地图轨迹图。
  - PocketBase 是否足以承担任务中枢，还是需要尽早拆出 worker。

## 待细化

- 各类风险的优先级、触发条件与缓解方案。
- 隐私模式、模糊起终点、公开分享默认设置。
- 成本模型，包括 API 调用、图像生成和文件存储。
- 监控、告警和运营反馈闭环。
