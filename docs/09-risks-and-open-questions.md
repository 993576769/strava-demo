# 风险与待决问题

## 目标

收敛当前最值得提前关注的风险和未决策事项。

## 当前技术风险

- Strava API 配额和授权失败可能影响同步体验
- 当前同步逻辑仍在接口内执行，活动规模扩大后会变慢
- 单进程任务 runner 适合 MVP，但扩展性有限
- S3 和 Doubao 都依赖真实环境变量，开发和部署配置容易出错
- 当前 schema 已定义，但还缺少完整的初始化、迁移和 seed 规范

## 当前产品风险

- 用户是否持续为“运动轨迹艺术化”能力留存，仍需验证
- Doubao 生成依赖真实第三方配额、网络和图片下载稳定性
- 分享能力未上线，传播性有限

## 当前开放问题

- 是否要尽快把 Strava 同步也改成后台任务
- 是否要把 art runner 拆成独立 worker
- 是否允许一个账号同时绑定 GitHub 和邮箱密码
- 结果页是继续强调下载，还是加入分享导向
- Prompt 模板是数据库管理还是代码版本化

## 待细化

- 成本模型
- 监控和告警
- 隐私脱敏策略
