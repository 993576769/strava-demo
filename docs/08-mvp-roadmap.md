# MVP 路线图

## 目标

给项目提供一个由近到远的推进顺序，优先完成最小可行版本，再逐步增强视觉质量、分享能力和平台能力。

## 当前结论

- 阶段一：项目重定位与基础清理
  - 替换模板默认的 `todos` 示例业务。
  - 建立文档、数据模型和新页面信息架构。
  - 准备 Strava 与 AI 服务接入的环境配置。
- 阶段二：完成产品登录与 Strava 连接
  - 跑通 GitHub 登录、Strava 连接、活动列表和活动详情。
  - 让用户能看到可用于生成的活动数据。
- 阶段三：完成首个可用的手绘图生成闭环
  - 发起生成任务。
  - 异步完成图片生成。
  - 在前端查看与下载结果。
- 阶段四：补齐账号体系与恢复能力
  - 增加邮箱/账号密码注册和登录。
  - 增加找回密码、重置密码和账号恢复流程。
  - 保持与 GitHub 登录一致的会话模型和用户资料模型。
- 阶段五：增强可分享性和作品体验
  - 增加分享页、模板样式、历史作品管理。
  - 优化生成质量、速度与失败恢复。
- MVP 验收口径建议：
  - 一个新用户能够完成 GitHub 登录。
  - 一个新用户能够完成 Strava 授权。
  - 至少一条带轨迹活动能够成功同步。
  - 用户能够发起生成并得到可下载的成品图。
- 后续阶段验收补充：
  - 用户可以用账号密码注册并登录到与 GitHub 登录同一套用户体系。
  - 用户可以独立完成找回密码和重置密码。

## 待细化

- 每个阶段预计工期与里程碑顺序。
- 首版必须交付的页面清单。
- GitHub 登录与账号密码体系的前端入口如何共存。
- Demo 数据与测试账号策略。
- 是否安排灰度测试和内测反馈周期。
